# Code of Conduct

This project follows a simple rule: be respectful and constructive.

## Expectations

- Assume good intent. Frame feedback around the work, not the person.
- Keep discussions on-topic and technical.
- Use inclusive language. The collection is community-maintained and open to contributors regardless of background.
- Disagreements are normal. Resolve them with reasoning and code, not personal attacks.

## Unacceptable behaviour

- Harassment, insults, or personal attacks.
- Posting credentials, tokens, or other secrets in public issues, PRs, or comments.
- Off-topic promotion, spam, or repeated derailing of discussions.

## Reporting

If you see behaviour that violates this code, contact the maintainers privately:

- Open a private security advisory on this repo, **or**
- Email `conduct@pleme.io` <!-- TODO: replace with real address before launch -->

Reports are read by the maintainers only and handled confidentially.

## Enforcement

Maintainers may, at their discretion, edit or remove comments, close issues or PRs, or ban contributors from the project. Decisions are explained in writing when feasible.

This code applies to all project spaces: GitHub repositories, discussion threads, and any project-affiliated chat or events.
