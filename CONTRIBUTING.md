# Contributing to `akeyless.akeyless`

Thanks for taking the time. This collection is **fully autogenerated** from
TOML resource specifications, so contributions look a little different
from a typical Ansible collection. Please read this page before opening
a PR — it'll save us both a round-trip.

> **Note.** This repo is a public mirror of
> [`ansible-akeyless-gen`](https://github.com/pleme-io/ansible-akeyless-gen).
> File issues and PRs against the upstream repo.

## Where to send what

| Change you want to make                              | Right repo                                              |
|------------------------------------------------------|---------------------------------------------------------|
| Add a brand-new module for a missing Akeyless API    | [`akeyless-terraform-resources`](https://github.com/pleme-io/akeyless-terraform-resources) — add a TOML spec under `resources/` |
| Fix a generator bug (wrong field type, missing mask) | [`ansible-forge`](https://github.com/pleme-io/ansible-forge) — emitter lives in `src/module_gen.rs` |
| Fix a parser bug (wrong IR for a given spec)         | [`iac-forge`](https://github.com/pleme-io/iac-forge)    |
| Improve examples, docs, issue templates, CI          | [`ansible-akeyless-gen`](https://github.com/pleme-io/ansible-akeyless-gen) — these are hand-maintained |
| Add a unit / integration test for a generated module | [`ansible-akeyless-gen`](https://github.com/pleme-io/ansible-akeyless-gen) — under `tests/` |

If you patch a `plugins/modules/*.py` file directly, your change will be
overwritten on the next regen. Always upstream the fix.

## Filing a missing-endpoint issue

The fastest path to a new module is to file an issue using the
[`Missing endpoint`](./.github/ISSUE_TEMPLATE/missing-endpoint.yml) template.
You'll be asked for:

- The OpenAPI `operationId` and a link to the upstream Akeyless API spec.
- Whether it's a CRUD resource, RPC action, or a list/get data source.
- One paragraph of use case.
- Optionally, a draft TOML spec you'd like us to start from.

If you've already drafted the TOML, open the PR directly against
`akeyless-terraform-resources` and tag this repo's issue from there.

## Adding a TOML spec (the "right" path for new modules)

A new resource lives in `akeyless-terraform-resources/resources/<name>.toml`.
The schema is documented in that repo's README. Minimum viable example:

```toml
[meta]
name = "my_thing"
shape = "resource"   # or "action" / "data_source"

[create]
operation_id = "CreateMyThing"

[read]
operation_id = "GetMyThing"
identifier_field = "name"

[update]
operation_id = "UpdateMyThing"

[delete]
operation_id = "DeleteMyThing"
```

Once merged upstream, run `iac-forge-cli generate --backend ansible` (or wait
for the next scheduled regen) and the module will appear here.

## Reporting a generator bug

Open a [bug report](./.github/ISSUE_TEMPLATE/bug.yml) here. Please include:

- The module name and the smallest playbook that reproduces.
- The Akeyless SDK version (`pip show akeyless`).
- The Ansible version (`ansible --version`).
- The full traceback or `register: result` dump.

Triage typically lands the fix in `ansible-forge` (emitter) within a week,
and the regen PR comes here as a follow-up.

## Improving examples, docs, tests

This is the most welcome class of contribution. Just send a PR — no upstream
dance required. A few notes:

- New examples go under `examples/` and should be self-contained and
  under 60 lines. Add a header comment block listing prerequisites,
  what's demonstrated, and expected output. Update `examples/README.md`.
- Unit tests under `tests/unit/` should exercise argument-spec shape and
  the `module_utils/akeyless_client.py` helpers. Integration tests under
  `tests/integration/targets/` are smoke tests that require live Akeyless
  credentials.
- Docs in `README.md` and `SECURITY.md` are hand-maintained — feel free to
  improve clarity.
